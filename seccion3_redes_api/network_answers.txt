-¿qué es HTTP?, 80 vs 443, por qué TLS. 

HTTP en la manera en que los navegadores se comunican a travez de los navegadores y servidore
donde se pueden hacer GET-POST, el cliente envía una petición pidiendo recursos
y el servidor decide responder un codigo de estado dependiendo de esto si se envía algún recurso.

Los puertos 80 Y 443 aunque uno parte del mismo protocolo, definimos el puerto 80 
comunicaión HTTP la cuál no esta cifrada caso contrario con el puerto 443 que utiliza el protocolo TLS
fundamental debido a que cifra toda la comunicación otorgando seguridad y confidencialidad a lso datos 
los cuales pueden ser interceptados

-Flujo Mínimo de Despliegue (CI/CD)

Estos procesos se llevan acabo bajo un grupo de desarrollo para garantizar la 
calidad y velocidad de software. De esta forma el Pipeline se estructura bajo etapas
entre ellas tenemos El control de versiones mediante git, las pruebas de integración y Despliegue
Los flujos se integran entre esto dos últimos respectivamente, CI detecta ,compilar Y
ejecuta pruebas , si se genera algún fallo se detiene el proceso, en cambio el CD se ejecuta
en el Despliegue una vez empaquetado el software se envia al entorno donde se puede verificar manualmente o
automatizada para su repectiva aprobación de calidad.
Herrmaienta CI:Github Actions
Herramienta CD:Docker
